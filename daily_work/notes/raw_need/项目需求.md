# 2606供应链订单监测智能体需求

**目标：** 在医药供应链管理系统中加入招采订单风险监测智能体，大致流程是根据设置的规则，预警异常订单，给出异常订单的详细证据、事件链、知识图谱，生成工单由相关人员处理
![[1.png]]

在带量采购前面加一个模块：智能体
逻辑：定规则→发预警→给证据→生成工单
使用人包括：

1. 一线业务员：每天看风险、处理工单的人
2. 区域经理：看自己省的情况、审批处置方案
3. 管理层：看数据总览、了解整体风险态势

第一步：定规则
监测指标（初步）包括：

1. 价格预警：订单中出现低价，低价会被国家监测到，然后联动到其他省份，影响药企价格体系，必须折算到单位可比价
规则1：低价采购预警
● 说明：针对某个品种，医院下订单的价格，低于客户设置的预警价
● 建议怎么处理：需要先核实以下几种情况（低价数据来源平台实际情况、预警价是不是及时更新了、配送企业、医院情况）；然后看证据链（该医院之前的下单价格，涉及的金额；再横向核实其他医院/其他省份的订单价格、负责配送企业其他订单价格）
**规则2：订单价差异常**
● 说明：同一个品规（折算至单位可比价）在最高价和最低价之间的价差在1.8倍（可自己设置）以上
● 建议怎么处理：核实各省价格数据是不是真实成交价
![[2.png]]
2. 配送响应：医院下了但配送企业拒绝配送、响应慢、或者配送率低（配送率=配送数量/采购数量）
规则1：拒绝响应预警
● 说明：某个订单的状态是部分退货、过期配送、经营企业拒绝、拒绝、拒绝配送、拒绝确认、拒绝入库、拒绝收货、拒绝响应、配送企业无法配送、企业拒绝配送、企业配送撤废、全部退货、缺货、退货、未及时配送、无法配送、已驳回、已拒绝、异议撤单、拒收。
● 建议怎么处理：联系配送企业问原因，评估要不要换一家，跟踪后续响应情况
规则2：响应不及时预警
● 说明：配送企业确认订单后，超过48小时还没发货
● 建议怎么处理：排查是不是季节性供货紧张，要不要调整配送计划
规则3：配送率低预警
● 说明：根据各省配送率要求设置配送率预警阈值，低于阈值则预警。配送率=（配送数量÷采购数量）
● 建议怎么处理：核实数据完整性，分析低配送率原因（断货？物流？），然后按照各省的考核周期定时推送异常订单（省份一般是按月度考核）
3. 终端丢失和新进：根据终端采购数量和采购周期计算预计库存消耗周期，超过时间未发生采购则判定为丢失；之前没有采购过（或丢失超过180天恢复采购）下订单、且采购数量超过一定值（自由设置）判定为有效新进，下单但是数额小判定为非自然流。
规则1：终端丢失预警
● 说明：比如某医院每一个月下一次单，每单300盒左右，根据平均采购数量和平均采购周期计算预计库存消耗周期，超过时间未恢复采购判定为丢失
● 建议怎么处理：联系医院问是不是换了药，分析不买的原因（价格？配送？竞品？），要不要做挽回方案
规则2：新进终端识别
● 说明：某医院首次买某药，且之前180天从没买过，并且采购数量超过一定值
● 建议怎么处理：确认是不是新开发的客户，评估采购潜力，跟踪后续采购稳不稳定
4. 销量波动：某一笔订单的采购量或采购频次跟平均水平比突然飙升或骤降
规则1：采购量异常波动
● 说明：某笔订单的采购量，超过该医院该药近6个月平均采购量的3倍；或者采购量与上月相比骤降
● 建议怎么处理：核实是不是合并下单，确认是不是在囤货，评估对库存和价格的影响
规则2：采购频次异常波动
● 说明：近30天的采购次数，超过近6个月平均频次的2倍；或者采购次数与上月相比骤降
● 建议怎么处理：看是不是季节性需求，确认采购模式有没有异常，结合终端丢失和新进一起看

页面：监测大屏

详细证据数据--价格偏离度（采购价和预警价的降幅）、低价过去出现的次数、涉及的采购金额、涉及的医院、配送企业（核查关联主体行为）

出现频率分析--近7天和14天的频次柱状图，附带趋势提示（比如近7天频率较前7天上升23%，建议重点关注）
关联线索--跟这条线索有关的其他线索，比如其他医院以同一个价格下单/或相同配送企业出现的低价订单
AI分析推荐--风险等级、分析结论、建议怎么处理
由AI根据以往处理经验生成推荐，比如产生低价订单了，首先要核查来源平台（有些省份经常会出现低价，客户遇到过相似案例），再看订单是不是有效（询问配送企业、医院），然后再根据出现的频次、金额，以往的处理经验，生成相关的操作建议。
