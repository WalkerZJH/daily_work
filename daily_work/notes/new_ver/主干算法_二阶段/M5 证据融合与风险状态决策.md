## 1. 模块定位

M5 是证据融合与风险状态决策模块。它不是新的训练模型，也不是新的概率融合模型。它只负责把前序模块输出整理成统一的候选状态。

M5 的输入包括：

```text
Stage 1 probability scorer
M1 candidate pool / business priority
M2 one-shot repeat propensity
M3 survival-lite refinement
M4 detector evidence
demand-shape guardrails
one-shot policy
```

M5 的输出不是线索卡，而是后续 `structured evidence bundle` 的结构化状态输入。

---

## 2. 核心原则

M5 必须保持概率语义隔离。

```text
churn_probability_H 仍来自 Stage 1；
business_priority_score_H 仍来自 M1；
repeat_probability_H 仍来自 M2；
survival_state 仍来自 M3；
detector_evidence 仍来自 M4；
M5 不重新训练模型，不重新计算概率。
```

禁止：

```text
1. 不得把 detector severity 加权成新的 churn probability；
2. 不得把 business_priority_score 解释为概率；
3. 不得把 one-shot repeat_probability 解释为 recurring churn_probability；
4. 不得让 LLM 参与状态判断；
5. 不得自动派单。
```

## 5. final_candidate_status 定义

建议枚举：

```text
priority_review
manual_review
observation_only
low_confidence_watch
one_shot_attention
not_actionable
```

### 5.1 priority_review

适用于：

```text
recurring 主候选；
business_priority_score 高；
survival_state 显示 materially_overdue 或 likely_churn_interval；
demand-shape 不限制为 observation-only；
detector 至少有一个有效证据，或 survival 证据足够强。
```

### 5.2 manual_review

适用于：

```text
业务排序较高；
但 survival_confidence 中等；
或 detector 证据不足；
或 demand-shape 要求人工复核。
```

### 5.3 observation_only

适用于：

```text
intermittent H3；
lumpy；
低频需求形态下 probability 高但证据不足；
demand-shape guardrail 明确禁止强预警。
```

### 5.4 low_confidence_watch

适用于：

```text
历史不足；
label_confidence_weight 低；
survival 使用 group prior fallback；
detector data_quality_status 不稳定。
```

### 5.5 one_shot_attention

适用于：

```text
one-shot high value 专项表；
repeat_probability / attention_score 仅用于 one-shot 关注；
不进入 recurring churn 解释。
```

### 5.6 not_actionable

适用于：

```text
缺少核心字段；
数据质量不足；
候选虽然入池但后续精查未支持预警。
```

## 6. review_priority 定义

`review_priority` 是人工复核优先级，不是概率。

建议枚举：

```text
P0
P1
P2
P3
```

### P0

```text
高 business_priority；
survival_state = likely_churn_interval 或 materially_overdue；
demand-shape 允许预警；
detector 证据存在或历史充分。
```

### P1

```text
business_priority 较高；
survival 有轻中度超期；
需要人工确认。
```

### P2

```text
观察对象；
低频采购关系；
证据不足但值得后续跟踪。
```

### P3

```text
one-shot attention；
或仅作为主页提示；
不作为强处置任务。
```

---

## 7. evidence_strength 定义

`evidence_strength` 只描述证据强弱，不是概率。

建议枚举：

```text
strong
medium
weak
insufficient
```

计算依据：

```text
survival_state
survival_confidence
detector_hit_count
detector_confidence
data_quality_status
demand_shape_guardrail
```

示例规则：

```text
strong:
  survival_state materially_overdue / likely_churn_interval
  且 survival_confidence >= 0.7
  且至少一个 detector hit confidence >= 0.6

medium:
  survival 有轻度或中度超期
  或 detector 有命中但 confidence 中等

weak:
  仅有 Stage 1 probability / business priority
  survival 或 detector 证据不足

insufficient:
  核心字段缺失
  或 demand-shape / data quality 禁止强解释
```

---

## 8. one-shot 处理规则

one-shot 不进入 recurring risk status。

M5 对 one-shot 只输出：

```text
final_candidate_status = one_shot_attention
review_priority = P2 或 P3
auto_dispatch_allowed = false
```

如果 `selected_attention_score` 很高，可设为：

```text
review_priority = P2
```

但必须保留：

```text
probability_reference = repeat_probability_H
probability_interpretation = first_purchase_repeat_probability_not_recurring_churn_probability
```

---

## 9. demand-shape guardrail 处理规则

M5 必须优先尊重 demand-shape guardrail。

```text
smooth:
  允许 priority_review

erratic:
  允许 manual_review / priority_review，但 H3 降置信

intermittent:
  H3 默认 observation_only
  H12 可 manual_review

lumpy:
  默认 observation_only
  除非 detector evidence strong，否则不进入 priority_review
```

---

## 10. M5 不做事项

M5 不做：

```text
1. 不训练融合模型；
2. 不更新 churn_probability_H；
3. 不重算 business_priority_score_H；
4. 不运行 detector；
5. 不调用 LLM；
6. 不生成线索卡；
7. 不自动派单。
```
