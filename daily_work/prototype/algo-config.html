L1主干
主干 P(alive) · BG/NBD
v1 启用 ▼
用途：把"采购节奏放缓"数学化成可校准的存活概率，是最干净的领先指标。P(alive)持续下行就意味着医院正在逐渐淡出，是其他所有探测器的打分基础。
输入字段
每家医院 RFM：最近购买时间（Recency）、历史购买频次（Frequency）、首购距今（T）
命中信号 / 输出
P(alive) 存活概率 + 主干风险分 = (1−P_alive)×100；P(alive)下行趋势斜率
适用需求形态
平稳波动间断块状
政策性断崖交L2处理，不让主干误判
假设边界
BG/NBD 假设非契约独立泊松。医院采购受集采/医保政策驱动时，政策性断崖须交L2掩码处理
参数表
参数名当前值说明建议范围
重拟合周期 BG/NBD模型对全体医院重新拟合的频率 月度–半年
P_alive走势窗口 取近N个巡检点斜率判断"下行"（个） 2–6
P_alive 低阈值 低于此值视为P_alive低（红级前提之一） 0.25–0.55
P_alive 中阈值上限 P_alive处于[低阈值, 此值]之间为"中"（橙级条件） 0.55–0.85
L0.5闸门
需求形态分类闸门 · Syntetos-Boylan
v1 启用 ▼
用途：强制前置——先把每个分析单元分成平稳/波动/间断/块状四种形态，再决定用哪套探测器。不分类就上探测器，间断型药品（比如季节性用药）会成为误报喷泉。
输入字段
下单时间序列 → 计算ADI（平均需求间隔）和 CV²（采购量变异系数平方）
路由规则
间断/块状型：禁用Mann-Kendall趋势检验，改用Croston/SBA期望补货点；块状型仅保留P(alive)+期望补货点
适用需求形态
全部形态 （此步骤本身就是形态判定入口，不可禁用）
注意
此模块为强制前置，不可关闭；调整ADI/CV²阈值会影响全量单元的探测器路由
参数表
参数名当前值说明建议范围
ADI 阈值 平均需求间隔(ADI)高于此值视为间断/块状型 1.1–1.6
CV² 阈值 采购量变异系数平方(CV²)高于此值视为波动/块状型 0.3–0.7
探测器1
采购节奏异常（IPI）
v1 启用 ▼
用途：距上次下单天数超出该医院历史正常间隔就报警。这是最直接的节奏异常信号——医院该下单了，但它没有，可能是在观望或已换供应商。
输入字段
下单时间、药品编码、医疗机构编码
命中信号 / 输出
鲁棒z分（距末次下单天数与历史中位MAD的偏差倍数），超Z_HIT命中，超Z_FULL满分
适用需求形态
平稳波动间断→Croston改版块状→Croston改版
置信度机制
c = min(n / N_CONF, 1)，历史订单数不足N_CONF单时置信度打折，不影响分级
参数表
参数名当前值说明建议范围
Z_HIT 命中阈值（≈80分位），超出即触发探测器 1.0–2.5
Z_FULL 满分阈值，z≥Z_FULL时严重度s=100 2.5–5.0
MAD_FLOOR MAD最小值兜底（天），防止极低波动单元误报 1–7
N_CONF 达到满置信度所需历史订单数（单） 8–20
探测器2
量/金额趋势衰减（Mann-Kendall + Theil-Sen + CUSUM）
v1 启用 ▼
用途：采购量持续单调下滑就报警。Mann-Kendall检验趋势显著性，Theil-Sen估算下滑斜率，CUSUM捕捉持续性均值下移——三种方法只要一个命中即触发。
输入字段
月度采购数量（去季节后）、采购金额、下单时间
命中信号 / 输出
MK τ值+p值、Theil-Sen斜率（盒/月）、CUSUM累积偏差C⁻；三者OR合并命中
适用需求形态
平稳波动 间断—禁用块状—禁用
间断/块状型禁用MK趋势检验，随机性会被误判为衰减
严重度计算
近3月相对下滑 drop = max(0, −slope×3/target)，s = min(100×drop/DROP_FULL, 100)
参数表
参数名当前值说明建议范围
MK_P Mann-Kendall检验p值阈值，小于此值视为显著下行趋势 0.05–0.2
CUSUM k CUSUM容忍参数（标准差的倍数），控制对轻微下移的敏感度 0.25–1.0
CUSUM H CUSUM触发阈值（σ倍数），C⁻ ≥ H×σ即触发 3–6
DROP_FULL 满分对应的相对下滑比例，跌60%记严重度100分 0.4–0.8
探测器3
采购频次衰减（泊松/负二项到达率）
v1 启用 ▼
用途：近窗口下单频率比历史基线明显下降就报警。频次下降往往比采购量下降更早出现，常是"医院开始减少下单频率但每单量还撑着"的早期信号。
输入字段
订单编号、下单时间（聚合为月均下单次数λ）
命中信号 / 输出
近窗口月均下单率λ_recent vs 基线期λ_base，rate_ratio = λ_recent/λ_base
适用需求形态
平稳波动间断 仅FREQ_MIN_BASE≥1月均单才启用此维度
置信度机制
c = min(λ_base/2, 1)，低频本身采购的单元此维度可信度低，不强行打分
参数表
参数名当前值说明建议范围
FREQ_DROP 频次降到基线的比例低于此值即命中（rate_ratio ≤ FREQ_DROP） 0.4–0.8
FREQ_MIN_BASE 基线期月均下单次数最低门槛，低于此值不启用频次维度（单/月） 1–3
探测器4
品规收窄 / 钱包份额
v1 启用 ▼
用途：在采品规数收缩是竞品逐个替代的最早前兆，常领先于总量下滑1–2个月。医院先停一个规格，再停另一个，最后全停——品规收窄是最早能抓到的那道裂缝。
输入字段
药品编码、规格、剂型（统计活跃品规distinct数）
命中信号 / 输出
shrink = (sku_base − sku_recent)/sku_base，≥SKU_SHRINK且基线≥SKU_MIN_BASE即命中
适用需求形态
平稳波动间断块状 本身单品规的医院此维度不适用（c自动趋0）
信号价值
品规逐个掉 = 竞品逐步替代，通常领先于总量下滑，是最早的可干预窗口信号
参数表
参数名当前值说明建议范围
SKU_SHRINK 品规收缩率阈值，≥1/3品规消失即命中 0.2–0.5
SKU_MIN_BASE 基线期活跃品规最低数量，低于此不启用此维度 2–4
探测器5
履约三段差（断供前兆）
v2 规划中 ▼
⚠️ v2 高优先级规划中 · 数据字段确认后上线 · 当前处于关闭状态
用途：用采购→配送→收货三段缺口，区分"医院不想买了（需求型流失）"和"医院想买但买不顺（断供型）"。这两类根因的干预动作完全不同：需求型找医院做客情，断供型找配送商修履约。混判会让一线跑错方向，是最贵的误报。
输入字段
采购数量×时间、配送数量×时间、收货数量×时间、配送企业编码
命中信号 / 输出
配送率fill_rate趋势下行 / 配送时延dlag拉长 / 拒单率reject_rate ≥ 阈值；输出断供/需求/双重根因标签
适用需求形态
平稳波动间断块状
核心产出
断供型：根因归配送商，不计入医院需求流失主因；配送商≥SYS_MIN_HOSP家同时恶化则独立成配送商预警
参数表
参数名当前值说明建议范围
REJECT_DAYS 超过N天无配送记录即判定为拒单（天） 5–14
REJECT_RATE_HIT 近90天拒单率达到此比例即命中 0.10–0.25
FILL_OK 配送率"健康线"，低于此值视为履约恶化 0.7–0.9
SYS_MIN_HOSP 配送商系统性失约阈值：≥N家医院同时履约恶化才生成配送商专项预警 3–10
探测器6
价格/竞争替代
v2 规划中 ▼
⚠️ v2 规划中 · 需跨院/跨省价格数据接入 · 当前处于关闭状态
用途：同一药品跨院/跨省采购价格离散度突变，往往领先于量价下滑，是竞品降价进院或本品涨价失去竞争力的早期信号。
输入字段
采购价格、制剂价格、批准文号（跨院/跨省汇总）
命中信号 / 输出
同药品跨院价格IQR/标准差突变，本院价格偏离同群中位，竞争替代预警标签
适用需求形态
平稳波动
数据依赖
需要同药品多院采购价格数据，单客户数据孤岛下效果受限；多客户汇聚后信号更准
探测器7
同侪对照（去伪）
v2 规划中 ▼
⚠️ v2 规划中 · 当前处于关闭状态
用途：按等级+地区分群，群体整体稳而单院特异下滑才算真风险。这个机制能压掉行业性普跌的假警报——如果全省三甲医院都在减少采购，那大概率是政策/季节因素，而非某一家医院的问题。
输入字段
机构等级（三甲/二甲等）、所属地区（省/市）
命中信号 / 输出
本院下滑分位 vs 同群中位；低于PEER_PCTL时视为群体共同下行，触发全局压制因子
适用需求形态
平稳波动间断块状
压制机制
suppress=True时：Score × SUPPRESS_F（默认0.4），即将风险分压到40%；作用于L3融合层
参数表
参数名当前值说明建议范围
PEER_PCTL 本院相对同群分位低于此值时，视为群体共同下行，触发压制 25%–50%
L2校正
去伪校正（集采掩码 / 季节扣除 / 断供分流）
v1 启用 ▼
用途：融合前先扣掉"假流失"——集采落标的合法量跌打掩码、季节囤货扣除、配送商系统性失约单独成线索而不冤枉医院。这一步是证据链可信度的基础。
输入字段
医保目录类型（集采/非集采/普通）、量价阶跃突变时间点
三类校正
① 集采落标→policy_masked打掩码不报；② 季节同比扣除；③ 配送商系统性失约→独立配送商线索，受影响医院流失分降权
适用需求形态
全部形态
注意
此模块强制启用，不可关闭；集采目录数据需与医保系统对接后才能充分发挥
此模块无可配置参数，由数据接口自动维护
L3融合
融合与印证升级（多族交叉验证）
v1 启用 ▼
用途：分数以主干P(alive)为准，探测器只做"是否多族印证"来影响分级，避免多个相关探测器叠加导致虚高分数（相关污染）。越多独立信号族同时命中，越可信。
印证系数公式
印证系数 = min(1 + ALPHA×(k−1), CORR_CAP)，k=独立命中信号族数；分数 = 主干分×印证系数×压制因子
信号族定义
F1=IPI节奏、F2=量趋势、F3=频次、F4=品规、F5=履约、F6=价格、F7=同侪；置信度c≥C_MIN才计入
适用需求形态
全部形态
封顶机制
CORR_CAP = 2.0，即无论多少族命中，印证系数最高翻倍，防止夸大
参数表
参数名当前值说明建议范围
ALPHA 每增加一个独立命中族的增益系数 0.3–0.8
CORR_CAP 印证系数封顶值，防止多族叠加时分数过度虚高 1.5–2.5
C_MIN 探测器最低置信度门槛，低于此值的命中不参与印证计数 0.3–0.6
SUPPRESS_F 同侪压制因子：被压制的单元分数乘以此系数 0.2–0.6
L3 FDR
FDR多重检验控制 · Benjamini-Hochberg
v1 启用 ▼
用途：每天对数千个「医院×产品线」单元跑多个统计检验，纯随机情况下也会"发现"成百上千条假风险。BH法把错误发现率（FDR）控制在目标线以下，只有经过FDR过滤后存活的命中，才算"独立命中信号族"参与印证升级。
工作原理
对当日所有探测器p值按升序排列，用BH逐步过程确定每个检验是否"存活"，控制期望假阳率 ≤ q
适用范围
全局作用：所有探测器的统计检验结果都经过此步骤，是信号质量的最后一道门
注意
FDR强制启用，不可关闭；降低q值可减少假阳率但会漏掉部分真实风险，应通过回测校准
适用需求形态
全部形态
参数表
参数名当前值说明建议范围
目标FDR q 期望错误发现率上限，q=0.10表示每天假风险不超过10% 0.05–0.20
L4分级
预警分级（红/橙/黄 印证逻辑）
v1 启用 ▼
用途：把风险分翻译成红/橙/黄三个等级，给一线一个可操作的优先级。分级不是纯阈值切分，而是基于"是否多族印证"的逻辑规则，避免单一指标偶发波动就触发高等级误报。
红级规则
P_alive低（<0.4）
且 ≥2个独立信号族（经FDR）
且 未被L2解释（非集采/季节）
橙级规则
P_alive中（0.4–0.7）
或 1个强信号族 + 同侪对照确认特异
黄级规则
单一信号族命中，P_alive未到低阈值；进入观察清单，暂不推到VP工作台
阈值可配置性
P_alive阈值应由回测校准后设定，不应脱离实际流失率数据随意调整
阈值在下方"分级阈值配置"区单独调整
L5校准
概率校准 · Platt Scaling / Isotonic Regression
v1 启用 ▼
用途：让风险分78真正对应约78%的流失率，否则分级阈值无意义。这是整个系统"说到做到"的校准层——处置反馈（兑现/未兑现）会持续回流，用于更新校准函数，使分数越来越名副其实。
校准方法
Platt Scaling（逻辑回归校准，数据量少时）或 Isotonic Regression（数据量足时，非参数单调校准）
质量指标
Brier分（综合准确度）、ECE期望校准误差（分箱校准偏差）、可靠性图（分布可视化）
数据来源
线索状态机中"声称挽回→兑现/未兑现"的处置结果，配合下一巡检周期的实际订单回填
注意
校准需要足够的标注数据（通常≥200条有结果的线索），v1初期校准效果有限，会随使用时间提升
校准参数由系统自动学习，无手动配置项

分级阈值配置

调整红/橙/黄分级的核心参数 · 修改后需点击保存 · 建议基于回测报告校准后调整
低于此值视为P_alive低，是红级必要条件之一；建议0.25–0.55
P_alive处于[红线, 此值]之间为"中"，是橙级条件之一；建议0.55–0.85
红级要求命中的独立探测器族数≥k（经FDR过滤）；默认≥2，降低会增加红级数量
每日批量检验的期望假阳率上限；0.10=每天红/橙中假风险≤10%
